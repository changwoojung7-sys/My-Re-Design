# 📢 [업데이트] v2.1.4 (Version Code 31) 배포 안내

안녕하세요, **My Re Design**입니다!
안드로이드 기기에서의 프리미엄 결제(PortOne) 오류를 수정하고 안정성을 높인 **v2.1.1** 업데이트를 배포했습니다.

### 💳 안드로이드 결제 (PortOne 외부 앱 호출) 오류 수정

#### 1. 안드로이드 외부 결제 앱 연결 문제 해결

* **문제**: 안드로이드(갤럭시 S25 등 안드로이드 11 이상) 기기에서 프리미엄 결제 시, 토스나 은행 앱카드 등 외부 결제 앱으로 화면이 전환되지 않고 웹뷰가 멈추는 오류(하얀 화면 또는 무반응)가 발생했습니다.
* **원인**: 최신 안드로이드 보안 정책으로 인해 외부 앱을 실행하려면 인텐트 스키마 권한(`queries`)이 명시되어야 하며, 하이브리드(Capacitor) 웹뷰에서는 `intent://` 형식의 외부 호출을 기본적으로 무시하기 때문이었습니다.
* **수정 사항**:
  * **보안 권한 추가**: `AndroidManifest.xml` 내에 한국 주요 은행 및 카드 앱 26종의 패키지 스키마 권한을 추가하여 외부 호출을 허용했습니다.
  * **결제 인텐트 가로채기 적용**: 안드로이드 웹뷰 브릿지(`MainActivity.java`) 설정을 오버라이드하여, 결제 모듈에서 외부 앱을 띄우는 신호를 캐치하여 시스템 브라우저나 결제 앱을 직접 구동하도록 구조를 보완했습니다.
  * **딥링크 복구 로직**: 결제가 끝난 후 앱으로 되돌아올 때 세션이 끊어지는 오류를 방지하기 위한 복구 로직(`onNewIntent`)을 추가 탑재했습니다.

### 🚀 버전 정보

* **Version Name**: `2.1.1`
* **Version Code**: `30`

> **[필독] 안드로이드 빌드 환경 관련 안내**
> 최신 버전의 원활한 빌드와 인텐트 권한 반영을 위해, Android Studio 내의 **Gradle JDK** 설정을 **Java 11 이상 (Java 17 권장)**으로 설정해주시기 바랍니다. (Settings > Build, Execution, Deployment > Build Tools > Gradle > Gradle JDK)

## 📢 [업데이트] v2.1.0 (Version Code 29) 배포 안내

안녕하세요, **My Re Design**입니다!
최신 안드로이드 기기 지원 및 빌드 안정성을 대폭 강화한 **v2.1.0** 업데이트를 배포했습니다.

### 🛠 빌드 시스템 및 호환성 최적화

#### 1. 최신 안드로이드 기기(갤럭시 S25 등) 지원 강화

* **문제**: 최신 Java 21 및 Android 15 환경에서 일부 라이브러리 호환성 문제로 인해 앱 설치가 실패하거나 빌드가 중단되는 현상이 있었습니다.
* **수정 사항**:
  * **Gradle 버전 최적화**: Java 21과 완벽히 호환되는 Gradle 8.12 버전으로 업그레이드했습니다.
  * **SDK 사양 조정**: `compileSdkVersion`을 36으로 상향하여 최신 라이브러리 요구사항을 충족하고, `targetSdkVersion`을 35로 유지하여 현재 최신 기기(S25)에서의 안정성을 확보했습니다.
  * **AGP(Android Gradle Plugin) 업데이트**: 최신 AndroidX 라이브러리가 요구하는 AGP 8.9.1로 빌드 도구를 현대화했습니다.

#### 2. 앱 실행 안정성 확보 (AdMob 크래시 해결)

* **문제**: 앱 실행 즉시 AdMob SDK 초기화 중 `Missing application ID` 오류로 인해 튕기는 현상이 발생했습니다.
* **수정 사항**: `AndroidManifest.xml`에 AdMob App ID 메타데이터를 정상적으로 복구하여 실행 안정성을 확보했습니다.

#### 3. 프로젝트 구조 및 리소스 복구

* **문제**: 빌드 과정 중 필수 리소스(아이콘, 테마, 매니페스트) 및 핵심 자바 클래스(`MainActivity`)가 유실되는 문제가 식별되었습니다.
* **수정 사항**:
  * 유실된 `MainActivity.java` 및 패키지 구조를 원상 복구했습니다.
  * `AndroidManifest.xml`, `strings.xml`, `styles.xml` 등 필수 리소스를 재생성하고 어댑티브 아이콘 설정을 추가했습니다.

### 🚀 버전 정보

* **Version Name**: `2.1.0`
* **Version Code**: `29` (기존 v2.0.8 대비 상향)

## 📢 [업데이트] v2.0.7 (Version Code 28) 배포 안내

안녕하세요, **My Re Design**입니다!
미션 생성 로직의 고도화와 기기 간 동기화 이슈를 완벽히 해결한 **v2.0.7** 업데이트를 배포했습니다.

### 🔄 미션 새로고침(Refresh) 동기화 완료

#### 1. 카테고리별 독립적인 미션 새로고침 카운팅

* **문제**: 모바일 기기와 웹, 혹은 안드로이드 기기 간 미션을 새로고침(변경)할 때 횟수가 동기화되지 않거나, 다른 카테고리의 횟수가 같이 차감되는 버그가 있었습니다. (로컬 스토리지 한계 및 카테고리 분류 오류)
* **수정 사항**:
  * 기기 내부 저장소(LocalStorage)에 의존하던 방식을 버리고, **Supabase DB(`mission_refresh_log`)를 직접 참조**하여 동기화하는 방식으로 서버-클라이언트 통신 구조를 전면 개편했습니다.
  * 서버의 Edge Function에서 `refreshCategory`를 정확히 식별하도록 구조를 개선하여, 이제 건강, 커리어 등 각 카테고리별로 하루 3회의 새로고침 기회가 **독립적이고 정확하게** 관리됩니다.
* **효과**: 어느 기기로 접속하든 정확한 잔여 새로고침 횟수를 확인할 수 있으며, 미션 변경이 실시간으로 동기화됩니다.

### 🎮 FunPlay 미션 대규모 확장

#### 2. FunPlay 미션 3개 동시 생성 및 패턴 확장

* **기존**: FunPlay 카테고리는 가벼운 미션 1개만 생성되며, 패턴(Archetype, Twist 등)이 5~10개 수준에 머물러 있어 며칠 후 반복되는 느낌이 있었습니다.
* **수정 사항**:
  * **미션 3개 생성**: 다른 핵심 미션들과 동일하게, FunPlay에서도 한 번에 **3개의 미션**을 제공하여 선택의 폭을 넓혔습니다.
  * **난이도 시스템 및 패턴 대폭 확장**: AI가 미션을 생성할 때 쓰는 패턴 라이브러리를 `Easy`, `Medium`, `Hard` 3단계 난이도로 분리했습니다.
  * Archetype(테마) **30개**, Mechanic(제약 조건) **37개**, Twist(변형) **30개**로 기존 대비 **10배 이상** 패턴 풀을 확장했습니다. (조합 가능 수 10,000개 이상)
* **효과**: 매일 전혀 다른 난이도와 테마의 신선하고 기발한 FunPlay 미션을 3개씩 받아보며 일상의 소소한 재미를 무한대로 즐기실 수 있습니다.

### 💸 무료 체험자(Trial) 미션 수익화 로직 개선

#### 3. 페이월(Paywall) 시점과 미션 생성 순서 교정

* **기존**: 무료 체험 Phase 4 단계에서 리워드 광고를 시청하지 않았음에도 AI가 백그라운드에서 미션을 미리 생성해버려 호출 자원 낭비 및 로직 꼬임이 있었습니다.
* **수정 사항**:
  * 미션을 서버에 요청하기 전에 페이월(광고/구독 확인) 방어막이 완벽히 작동하도록 가드(Guard) 로직(`shouldBlockForPaywall`)을 추가했습니다.
  * **광고 시청을 완료하거나 구독을 한 직후**에만 미션이 생성되도록 순서를 바로잡았습니다.
